# 辅助手册

## 项目定位
- 这是一个个人网站项目，用来承载随笔、摄影、音乐推荐、感受随想、珍贵回忆、日记与灵感。
- 当前技术栈是 Hexo + Butterfly 主题。

## 当前已完成的重要能力

### 1. 基础站点内容与导航
- 已创建首篇站点规划文章，用于说明网站后续装点计划。
- 已补齐标签页与分类页，解决过 `Cannot GET /tags/` / 分类页不可访问的问题。
- 首页与常规页面的侧边栏能力已按当前主题结构接入。

### 2. DOCX 随笔自动导入
- `大澪神神狂想集！` 文件夹中的 docx 会被自动导入为文章。
- docx 是唯一内容源，文章内容由导入流程生成，不手工维护镜像正文。
- 已支持：
  - 构建时自动导入
  - 开发时实时监听更新
  - 导入 docx 内图片
- 狂想集文章统一归档到“大澪神神狂想集”标签。
- 时间规则：
  - 发表时间 = 该 docx 首次被导入时的时间
  - 更新时间 = docx 最近一次文件修改时间
- 链接形式已经从早期 hash 风格调整为更好看的中文/拼音风格方案。

### 3. 阅读体验优化
- 狂想集相关文章的正文字体已整体调大。
- 页面中已加入字体调节滑动条，用于提升阅读舒适度。

### 4. 站点展示策略
- 站点总访问数与总浏览量不再展示。
- 每篇文章仍保留真实浏览量显示。

### 5. 对外部署与页面入口
- 已确定当前对外部署方案为 GitHub Pages，并通过 GitHub Actions 自动构建发布。
- `themes/butterfly` 已改为主仓库内的普通目录，不再维持错误的子模块状态。
- 仓库已明确不追踪 `node_modules/`、`public/`、`db.json`、`.deploy_git/` 等构建产物或本地依赖。
- 文章底部原先的“分享按钮”已取消，不再沿用分享组件逻辑。
- QQ、哔哩哔哩、小红书入口已移动到右侧边栏头像下方。
- QQ 入口已改为站内页面 `/contact/qq/`，页面仅保留二维码与强调文案，不再显示额外说明、昵称或号码。

## 已明确回退的功能

### 本地文件浏览
- 这个功能曾支持通过网页浏览本机白名单目录，并尝试加入图片/视频预览。
- 现已决定整体回退。
- 回退原因：
  - 实现与维护成本高
  - 与个人站点的核心价值相比优先级不够高
  - 面对大量视频目录时，当前架构下性能和体验都不理想
- 后续如再考虑类似能力，应从“是否值得作为独立工具存在”重新评估，而不是直接恢复旧实现。

## 关键项目事实
- 这个项目更偏向“长期个人内容站”，核心价值在内容沉淀与表达，而不是重型工具化能力。
- 做新功能时，优先判断它是否直接服务于内容展示、阅读体验、个人表达与回忆整理。
- 如果目标明确但路径过重，应主动提出更短路径。

## 与 Claude 协作约定
- 用户希望用第一性原理讨论问题：
  - 不要默认用户已经完全清楚目标和最优路径。
  - 如果目标不清晰，要先讨论动机和需求。
  - 如果目标清晰但当前方案不是最短路径，要明确指出并建议更好的办法。
- 对项目层面的重要新增、回退、设计决定，应更新本手册，方便新对话快速恢复上下文。
